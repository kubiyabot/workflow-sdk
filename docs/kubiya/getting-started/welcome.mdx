## Why We Built This

After watching teams struggle with free-wheeling agent frameworks that promise magic but deliver chaos, we took a different approach. Instead of hoping an AI will figure out the right sequence of actions, we provide the tools to **define** the right sequence ‚Äì with AI filling in the intelligent parts.

## Quick Example

<CodeGroup>
  ```python Basic Workflow
  from kubiya_workflow_sdk import Workflow, Step, Client

  # Define a deterministic workflow
  workflow = Workflow(
      name="deploy-app",
      description="Deploy application with rollback",
      steps=[
          Step(
              name="validate",
              command="kubectl apply --dry-run=client -f deploy.yaml"
          ),
          Step(
              name="deploy",
              command="kubectl apply -f deploy.yaml",
              depends=["validate"]
          ),
          Step(
              name="health_check",
              command="kubectl wait --for=condition=ready pod -l app=myapp",
              depends=["deploy"],
              retry={"limit": 5, "interval_sec": 30}
          )
      ]
  )

  # Execute with real-time streaming
  client = Client()
  for event in client.execute_workflow(workflow, stream=True):
      print(f"{event.type}: {event.data}")
  ```

  ```python AI-Powered Generation
  from kubiya_workflow_sdk.providers import get_provider

  # Use ADK provider to generate workflows from natural language
  adk = get_provider("adk")

  # Generate and execute in one go
  result = await adk.compose(
      task="Deploy my app to production with health checks and rollback capability",
      mode="act",  # Generate AND execute
      stream=True
  )

  # The AI generates a complete workflow using the SDK
  # with proper error handling, retries, and monitoring
  ```
</CodeGroup>

## üéØ Key Features

### üèóÔ∏è Composable Building Blocks

<AccordionGroup>
  <Accordion title="Workflow DSL" icon="code">
    Pythonic interface for defining workflows with a fluent API
    ```python
    workflow = (
        workflow("my-automation")
        .description("Automate all the things")
        .params(ENV="${ENV}", VERSION="${VERSION}")
        .step("build", "docker build -t app:${VERSION} .")
        .step("test", "pytest tests/", depends=["build"])
        .step("deploy", "kubectl apply -f k8s/", depends=["test"])
    )
    ```
  </Accordion>

  <Accordion title="AI Agent Integration" icon="brain">
    Embed AI intelligence where it matters
    ```python
    .step("analyze")
    .inline_agent(
        message="Analyze deployment risk for ${VERSION}",
        agent_name="risk-analyzer",
        ai_instructions="Evaluate changes and provide risk assessment",
        runners=["production-runner"]
    )
    .output("RISK_LEVEL")
    ```
  </Accordion>

  <Accordion title="Tool Ecosystem" icon="toolbox">
    Define and use containerized tools inline
    ```python
    .step("notify")
    .tool_def(
        name="slack-notifier",
        type="docker",
        image="kubiya/slack:latest",
        content="#!/bin/sh\ncurl -X POST $WEBHOOK -d '{\"text\":\"$1\"}'",
        args=[{"name": "message", "type": "string"}]
    )
    .args(message="Deployment complete!")
    ```
  </Accordion>
</AccordionGroup>
